## Project

Static PDF ebook library ‚ÄĒ Astro static site deployed to GitHub Pages, PDFs hosted on GitHub Releases. Personal public-welfare project. v1: functional, no complex UI.

## Commands

```bash
npm run dev          # Astro dev server (no Pagefind ‚ÄĒ search won't work locally)
npm run check        # astro check ‚ÄĒ TypeScript diagnostics
npm run validate     # tsx scripts/validate-books.ts ‚ÄĒ validates all book entries
npm run test:reading # python scripts/test-reading-metrics.py ‚ÄĒ reading-metric unit tests
npm run build        # astro build && pagefind --site dist ‚ÄĒ production build
npm run preview      # astro preview ‚ÄĒ serves dist/ locally
```

## Architecture

### Data flow

```
PDF (GitHub Release)
  ‚Üí book_assets.py (PyMuPDF) generates:
      public/covers/{id}_v{edition}.png         ‚ÄĒ first-page cover image
      public/covers/{id}_v{edition}_spine.png   ‚ÄĒ 1px-wide spine sample
      src/data/outlines/{id}_v{edition}.json    ‚ÄĒ PDF bookmarks
      src/data/reading/{id}_v{edition}.json     ‚ÄĒ page count, character counts, estimated minutes

Markdown books (src/content/books/*.md)
  ‚Üí Astro Content Collection (config.ts ‚ÄĒ Zod schema)
    ‚Üí books.ts (getAllBooks, getBookById, getBooksByClassification)
      ‚ĒĒ‚ĒÄ resolves covers (explicit ‚Üí generated ‚Üí placeholder), loads reading metrics
      ‚Üí Astro pages (index, books/[id], categories/[classification])
        ‚Üí Components (BookCover, BookCard, BookMeta, DownloadButton, ReadingStats, Outline)
```

### Call number system (the core abstraction)

Every book has a call number `id` like `A12-8-2`. `src/lib/bookId.ts` parses it and **derives everything**:

- `A12-8-2` ‚Üí classification `A12`, accession 8, volume 2, workId `A12-8`
- `A8-3` ‚Üí classification `A8`, accession 3, no volume, workId `A8-3`
- Regex: `/^[A-Z](?:\d+(?:\.\d+)?)?-\d+(?:-\d+)?$/` ‚ÄĒ supports `A8-3`, `A12-8-2`, `I210.4-1`, `T-1`, `Z228-1`

**Never hand-write** `download_url`, `release_tag`, `pdf_filename`, `classification`, or `volume` in book markdown. These are all derived from `id` + `edition` by `bookId.ts` (+ `books.ts` for the GitHub URL using `siteConfig`).

### Cover system

`books.ts` resolves covers in 3 tiers:

1. **Explicit** (`coverKind: "explicit"`) ‚ÄĒ book frontmatter has a `cover` field pointing to a custom image
2. **Generated** (`coverKind: "generated"`) ‚ÄĒ `public/covers/{id}_v{edition}.png` exists on disk (generated by `book_assets.py`)
3. **Placeholder** (`coverKind: "placeholder"`) ‚ÄĒ no image available; `BookCover.astro` renders a typographic fallback

Spine images (`{id}_v{edition}_spine.png`) are 1px-wide samples of the first page's leftmost pixel column, used as `--spine-texture` in the 3D book model. A `pages-texture.svg` at `public/images/` provides the page-edge texture.

### Reading metrics

`books.ts` loads `src/data/reading/{id}_v{edition}.json` (generated by `book_assets.py`) with this structure:

```json
{
  "page_count": 320,
  "cjk_character_count": 85000,
  "latin_token_count": 1200,
  "estimated_minutes": 288
}
```

Reading time is determined by two independent rates from `reading-config.json`:
- CJK: characters √∑ `cjk_chars_per_minute`
- Latin: tokens √∑ `latin_words_per_minute`
- Sum of both, rounded up (minimum 1)

A `readtime` field in book frontmatter overrides the automatic estimate (`readingTimeSource: "manual"`). Character counts are only stored when `units_per_page ‚Č• sparse_text_units_per_page` (to skip image-heavy PDFs).

`ReadingStats.astro` renders page count, approximate character count, and estimated minutes (prefixed with "Áļ¶" when automatic).

### Outline JSON

Located at `src/data/outlines/{id}_v{edition}.json`. Structure: `[{ level, title, page }]`. Loaded at build time via `fs.readFileSync` in `[id].astro`. Empty array `[]` means no bookmarks ‚ÄĒ the Outline component won't render anything.

### Pagefind

Runs as a post-build step (`pagefind --site dist`). The search page (`search.astro`) loads Pagefind UI JS/CSS from the built `dist/pagefind/` at runtime. In dev mode (`astro dev`), search silently does nothing ‚ÄĒ that's expected.

### Base path

All internal links must use `import.meta.env.BASE_URL` (resolves to `/ebook-library/` in production). The pattern is `` `${base}/path/` `` ‚ÄĒ note the `/` separator between base and path. Astro config sets `base: "ebook-library"` (no leading/trailing slash ‚ÄĒ Astro normalizes it).

### Styling

Single global stylesheet (`src/styles/global.css`) with CSS custom properties. No CSS framework, no dark mode. Three responsive breakpoints (760px, 560px, 340px) for layout adjustments on small screens. The detail page renders a 3D book model via CSS transforms (`skewY`, `rotate`, `translate`) with custom properties `--pages-texture`, `--spine-texture`, and `--book-size`. Hover animations respect `prefers-reduced-motion`.

## Adding a book

### First release

1. Determine call number (e.g., `A12-8-2`)
2. Compile PDF, name it `A12-8-2_v1.pdf`
3. Create GitHub Release with tag `A12-8-2_v1`, upload PDF
4. Create `src/content/books/A12-8-2.md` with frontmatter (id, title, edition, date, tags ‚ÄĒ author/cover/total_volumes/readtime optional)
5. Generate static assets from the PDF:
   ```bash
   # From a GitHub Release (needs gh CLI + PyMuPDF):
   ./scripts/extract-outline-from-release.sh A12-8-2 1

   # Or from a local PDF file:
   pip install PyMuPDF
   python scripts/extract-book-assets.py path/to/A12-8-2_v1.pdf A12-8-2 1
   ```
   This produces: cover PNG, spine PNG, outline JSON, and reading metrics JSON.
6. Commit the .md and all generated assets, push to `main`
7. GitHub Actions deploys automatically

### New edition

1. Bump `edition` in the book's .md (e.g., 1 ‚Üí 2)
2. Name the new PDF `A12-8-2_v2.pdf`
3. Create Release with tag `A12-8-2_v2`, upload PDF
4. Re-run asset extraction for the new edition
5. Commit updated .md + new assets

## Constraints (v1)

- **No** database, user accounts, comments, CMS, SSR, online PDF reader
- **No** EPUB/MOBI/HTML formats ‚ÄĒ PDF only
- **No** dark mode, complex UI
- **No** About page, standalone copyright page
- **No** hierarchical classification browsing ‚ÄĒ `/categories/A12/` only shows exact `A12-*` books, not subcategories
- **No** outline-to-PDF page linking ‚ÄĒ outline is display-only
- **No** GitHub Release existence check at build time (offline-safe)
