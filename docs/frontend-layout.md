# 前端排版合同

**适用对象：** 首页、书籍详情页和响应式布局维护者。

## 七字宽书名

`src/lib/bookTitleLayout.ts` 只判断主标题是否少于七个 Unicode 字符，不决定具体
像素宽度。首页和详情页复用这一判断，但各自按照实际字体字面设置宽度：

- 首页使用普通锦书宋，汉字进宽为 `1em`，短标题铺满 `7em`；
- 详情页使用扁字形锦书宋，汉字进宽为 `0.75em`，短标题铺满 `5.25em`；
- 七字及以上标题保持自然排版，不强制压缩；
- 单字标题在对应七字宽区域内居中；
- 不使用 `letter-spacing` 模拟分布，避免把字距叠加到字形宽度上。

首页和详情页不得共享固定的 `7em` 或 `5.25em` 数值。更换字体时应先读取字体实际
advance width，再分别调整视图样式；不能根据视觉印象假定字体是全角或扁字形。

## 简讯分段符

置顶消息和普通消息同时存在时，简讯栏显示三个全角星号 `＊`。分段符宽度始终为
消息内容 frame 的 `100%`，内部使用三列等分网格，每个星号在本列居中。三个中心
相对于 frame 左侧的位置应分别为 `1/6`、`1/2`、`5/6`。

不得通过固定像素宽度、左右 padding 或手工空格排列星号。frame 在响应式布局中变化
时，分段符必须同步伸缩。

## 响应式验收

至少检查以下视口：

- 桌面：`1440 × 900`
- 横向平板：`1024 × 768`
- 竖向手机：`390 × 844`
- 窄屏保护：`320 × 568`

等待 `document.fonts.ready` 后再测量文字。短标题应使用对应视图的字体与七字宽；
简讯分段符应与内容 frame 同宽；页面不得出现横向溢出。打开带正文的简讯后再返回，
列表位置、原触发按钮和布局宽度应恢复。

代码修改后运行：

```bash
mise exec -- npm run test:ui
mise exec -- npm run check
mise exec -- npm run build
```

合并前仍以 `mise exec -- npm run verify` 为完整验证入口。
