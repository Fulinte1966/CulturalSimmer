# 贡献指南

## 开始前

1. 从 `main` 创建独立分支。
2. 使用 `mise install` 和 `mise run setup` 安装仓库声明的 Node、Python 与依赖。
3. 阅读 [文档索引](docs/README.md) 中与改动对应的契约和流程。

## 改动边界

- 不手写 PDF 入库产生的书目派生字段或下载地址。
- 不把 PDF、凭据、日志、备份、完整设计导出或智能体配置加入 Git。
- 不在无关功能改动中重排生成数据或大体积字体文件。
- 人工公告以 `src/content/announcements/*.md` 为源；更新归档由脚本生成。
- 可见文字变化需要重新生成字体子集并检查浏览器渲染。

## 验证

常规提交运行：

```bash
mise run check
```

该命令包含字体双遍构建。只排查字体生成器时才单独运行 `mise run fonts`。

涉及 PDF 入库、版本差异、更新 feed 或 Release 时，应按对应文档增加定向测试，并在测试仓库或经授权的正式链路中完成一次端到端验证。

## Pull Request

Pull Request 应说明改动原因、用户影响、验证命令和剩余风险。不要把秘密值、原始运行日志或本地绝对路径写入描述。
